# 租费异常分析工具设计

## 背景

当前项目已经新增电费压降分析模块，并建立了通用专题输出表 `analysis_opportunities`。铁塔租费台账现有稽核规则已覆盖重复计费、共享用户数异常、共享折扣异常、原产权方电力引入费、停租后计费、挂高塔高异常、产品属性异常等问题。

本次功能继续保持本地单机版形态，不新增联网、多用户或外部系统依赖。目标是在现有规则和电费专题框架之上，新增“租费异常分析”模块，把租费稽核问题整理成业务人员更容易处理的异常线索清单。

## 命名口径

- 页面名称：租费异常分析。
- 表格名称：异常线索清单。
- 确定性金额字段：预计可追回金额。
- 优惠类金额字段：优惠落实金额。
- 暂不能明确归属的金额字段：待核查金额。

不使用“机会清单”作为页面或表格主名称，避免业务人员误解为确定收益或商业机会。

## 目标

第一版目标是让业务人员在导入并稽核铁塔租费台账后，可以进入“租费异常分析”页面，查看租费异常总览、异常分类、地市排行和异常线索清单，并导出 Excel 给地市、专业线或铁塔公司核查。

第一版强调：

- 复用 `analysis_opportunities`，通过 `domain = "tower_rent"` 区分专题。
- 复用现有稽核问题，不重写租费规则引擎。
- 优先覆盖能形成金额线索的异常。
- 对共享折扣、共享用户数等需合同或确认单复核的情况，明确标记为待核查或优惠落实。

## 非目标

- 不做联网协同或多用户审批。
- 不替代现有“稽核结果”和“问题包导出”流程。
- 不把待核查金额表述为确定损失。
- 不在第一版引入复杂租费计价模型或铁塔合同自动解析。
- 不要求第一版自动完成与铁塔系统的账单对账。

## 用户流程

1. 业务人员按现有流程导入台账并执行稽核。
2. 进入“租费异常分析”页面。
3. 点击“生成分析”，系统从当前批次租费稽核问题生成异常线索。
4. 查看总览指标、异常分类和地市排行。
5. 在“异常线索清单”中查看站址、账期、异常类型、金额口径、置信度和建议动作。
6. 导出“租费异常线索清单” Excel，用于地市核查、合同复核或铁塔账单沟通。

## 功能范围

### 总览

页面展示以下指标：

- 租费台账记录数。
- 涉及站址数。
- 年化租费总额。
- 异常站址数。
- 异常线索数量。
- 预计可追回金额合计。
- 优惠落实金额合计。
- 待核查金额合计。

### 异常分类

第一版支持以下异常类型：

- 重复计费：产品服务费、维护费、场地费、电力引入费重复计费。
- 停租仍计费：停租日期已填仍发生费用，或账期晚于停租月份仍发生费用。
- 产品单元异常：产品单元数为 0 但费用非 0。
- 原产权方费用异常：原产权方站址仍收取电力引入费。
- 共享折扣异常：维护费共享折扣非最优惠。
- 共享用户数异常：铁塔共享用户数或机房共享用户数在同站址、同产品下不一致。
- 产品属性异常：同业务确认单前后账期铁塔产品不一致。
- 基础属性异常：挂高超过塔高、同站址塔高不一致。
- 无站址仍付费：站址主数据不存在但租费台账仍有正向费用。
- 金额环比突变：同站址同费用字段相邻账期金额突增或突降。

### 异常线索清单

每条线索包含：

- 线索编号。
- 批次 ID。
- 地市、区县。
- 电信站址编码、站址名称。
- 铁塔站址编码、铁塔站址名称。
- 账期。
- 异常类型。
- 风险等级。
- 当前金额。
- 参考金额。
- 预计可追回金额。
- 优惠落实金额。
- 待核查金额。
- 置信度。
- 来源规则。
- 来源台账 sheet 和原始行号。
- 问题说明。
- 建议动作。

## 金额口径

### 预计可追回金额

用于相对确定的费用异常：

- 重复计费：按重复费用项金额估算。
- 停租仍计费：按停租后仍发生的租费金额估算。
- 产品单元为 0 但费用非 0：按对应产品服务费估算。
- 原产权方电力引入费非 0：按电力引入费金额估算。
- 无站址仍付费：按租费台账正向费用合计估算。

### 优惠落实金额

用于需要核对合同、共享关系或政策口径的优惠类异常：

- 维护费共享折扣非最优惠：按当前维护费与参考折扣费用差额估算。参考折扣优先使用同站址共享场景下更低折扣；样本不足时先按 0 计入待核查金额。
- 共享用户数异常：第一版不直接估算确定优惠金额，优先计入待核查金额；后续可接入合同计价参数后转为优惠落实金额。

### 待核查金额

用于存在金额影响但第一版无法可靠归属的异常：

- 共享用户数异常。
- 产品属性异常。
- 基础属性异常。
- 金额环比突变。
- 缺少可解析金额字段的重复计费或停租计费。

待核查金额代表核查范围，不等同于确定可追回金额。

## 技术设计

### 数据模型

复用 `analysis_opportunities`：

- `domain = "tower_rent"`。
- `recoverable_amount` 对应预计可追回金额。
- `saving_opportunity_amount` 对应优惠落实金额。
- 第一版暂不新增数据库字段，待核查金额由服务返回和导出时计算为 `current_amount - recoverable_amount - saving_opportunity_amount` 的非负值。

若后续多个专题都需要显式存储待核查金额，再单独迁移新增字段。

### 分析服务

新增租费分析服务模块，建议命名为 `tower_rent_analysis.py`，职责：

- 校验批次存在、已稽核、包含租费台账。
- 读取租费台账、原始字段和现有稽核问题。
- 将租费相关 `issues` 转为 `analysis_opportunities`。
- 计算当前金额、预计可追回金额、优惠落实金额和待核查金额。
- 写入或刷新 `domain = "tower_rent"` 的专题结果。
- 返回总览、分类、地市排行和异常线索清单。
- 导出 Excel。

每次运行分析前，删除该批次 `domain = "tower_rent"` 的旧记录，保证分析结果可重复生成。

### 接口

新增本地 API：

- `POST /api/batches/{batch_id}/tower-rent-analysis/run`
- `GET /api/batches/{batch_id}/tower-rent-analysis/summary`
- `GET /api/batches/{batch_id}/tower-rent-analysis/opportunities`
- `POST /api/batches/{batch_id}/tower-rent-analysis/export`

机会清单接口继续返回字段名 `opportunities`，以减少前端和后续专题复用成本；页面上显示为“异常线索清单”。

### 前端

新增左侧导航“租费异常分析”。页面结构与电费专题保持一致：

- 顶部操作区：当前批次、生成分析、导出 Excel。
- 总览卡片：租费总额、异常站址、预计可追回金额、优惠落实金额、待核查金额。
- 异常分类：按异常类型展示数量和金额。
- 地市排行：按预计可追回金额、优惠落实金额、待核查金额排序。
- 异常线索清单：支持异常类型、风险等级、置信度筛选。

### 导出

导出文件名包含批次编号和“租费异常线索清单”。Excel 包含：

- 填写说明。
- 异常线索清单。
- 地市汇总。
- 异常分类汇总。

## 错误处理

- 批次不存在：提示“批次不存在”。
- 缺少租费台账：提示“当前批次没有铁塔租费台账，无法生成租费异常分析”。
- 尚未稽核：提示“请先执行稽核，再生成租费异常分析”。
- 批次已归档：允许查看和导出历史分析结果，不允许刷新分析。
- 金额字段缺失或不可解析：金额按 0 处理，并将线索保留为待核查。

## 测试

后端测试覆盖：

- 生成租费异常线索。
- 重复计费、停租仍计费、原产权方费用异常的预计可追回金额。
- 共享折扣异常的优惠落实金额或待核查金额。
- 重复运行刷新旧结果。
- 总览和筛选接口。
- Excel 导出 sheet 和表头。
- 批次不存在、未稽核、无租费台账等错误。

前端测试或静态检查覆盖：

- 新导航入口加载。
- 页面模块语法检查。
- 空状态展示。
- 生成分析、筛选和导出按钮调用正确 API。

## 分期

第一期：

- 新增租费分析服务。
- 新增租费专题 API。
- 新增“租费异常分析”前端页面。
- 新增租费异常线索 Excel 导出。
- 复用现有租费稽核规则生成线索。

第二期：

- 抽象电费和租费共用的专题分析查询、汇总和导出 helper。
- 增强共享折扣、共享用户数和续签优惠金额估算。
- 增加租费环比趋势和高金额站址排行。

第三期：

- 接入专题闭环状态。
- 将租费异常分析纳入归档报表。
- 支持按合同、产品类型和费用项做更细颗粒度分析。
