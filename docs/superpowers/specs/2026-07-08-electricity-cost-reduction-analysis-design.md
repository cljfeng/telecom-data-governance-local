# 电费压降分析工具设计

## 背景

当前项目是本地单机版基础数据治理工具，已经支持站址、铁塔租费、电费、发电费台账导入、稽核、问题包导出、整改回传和归档。现有电费规则已经覆盖重复报账、用电量突变、电价异常、零电量有费用、金额计算偏差、转供电无合同等问题。

本次功能不改变单机版部署方式，也不重做基础治理流程。目标是在现有批次、台账、规则和导出能力之上，新增一个业务化的电费专题分析模块，把零散稽核命中整理成可执行的电费压降机会清单。

## 目标

第一版目标是让业务人员在导入并稽核电费台账后，可以进入“电费压降分析”页面，查看电费异常总览、异常分类、重点地市和站址，并导出可供地市核查的压降机会清单。

机会金额分为两类：

- 可追回金额：相对确定的问题金额，适合进入强核查和追回流程。
- 压降机会金额：疑似优化空间金额，适合进入运营压降、合同谈判、现场核查或能耗治理流程。

第一版强调“能看、能算、能导出”，不引入复杂预测模型，不改变现有整改闭环主流程。后续铁塔租费分析复用同一套专题分析框架。

## 非目标

- 不做联网版、多用户权限或中心化平台。
- 不替代现有稽核结果页面和整改回传流程。
- 不把疑似机会金额表述为确定损失金额。
- 不要求第一版具备完整机器学习、自动归因或外部系统对接能力。

## 用户流程

1. 业务人员按现有流程导入台账并执行稽核。
2. 进入“电费压降分析”页面。
3. 查看批次电费总额、异常站址数、可追回金额、压降机会金额和重点地市。
4. 按异常类型查看清单，例如重复报账、高电价、异常增长、转供电异常、疑似空载。
5. 导出电费压降机会清单 Excel，发给地市或专业线核查。
6. 地市核查后，第一版可先线下反馈；后续再接入现有整改回传闭环。

## 功能范围

### 总览

电费压降分析页面展示以下指标：

- 电费台账记录数。
- 涉及站址数。
- 电费总额。
- 异常站址数。
- 可追回金额合计。
- 压降机会金额合计。
- 高风险机会数量。
- 按地市汇总的机会数量和金额。

### 异常分类

第一版支持以下电费异常分类：

- 重复报账：同站址、同电表、同账期出现重复缴费记录。
- 时段重叠：同站址、同电表抄表周期存在重叠。
- 零电量有费用：用电量为 0 但存在正向费用。
- 金额计算偏差：电费金额与用电量、电价、分摊比例计算值偏差超过阈值。
- 包干重复报账：包干站址仍标记报账。
- 高电价：电价高于配置阈值或明显偏离同区县同供电方式中位数。
- 异常增长：用电量或电费金额环比异常增长。
- 转供电异常：转供电无合同、转供电价格偏高或缺少合同依据。
- 疑似空载：低电量或零电量状态下持续产生费用，或停用、退网站址仍发生电费。

### 机会清单

机会清单每条记录包含：

- 机会编号。
- 批次 ID。
- 地市、区县。
- 电信站址编码、站址名称。
- 电表户号。
- 账期或抄表周期。
- 异常类型。
- 风险等级。
- 当前金额。
- 参考金额。
- 可追回金额。
- 压降机会金额。
- 置信度。
- 建议动作。
- 来源规则。
- 来源台账 sheet 和原始行号。

置信度分为高、中、低。确定性强的重复报账、零电量有费用、金额计算偏差默认置信度更高；依赖统计参考值或业务假设的高电价、异常增长、疑似空载默认置信度为中或低。

## 金额口径

### 可追回金额

可追回金额只用于相对明确的问题：

- 重复报账：同组重复记录中，按疑似重复部分的电费金额估算。
- 时段重叠：按重叠记录金额或重叠天数比例估算，缺少起止日期时按较小金额保守估算。
- 零电量有费用：按正向电费金额估算。
- 金额计算偏差：按填报金额与计算金额的正向差额估算。
- 包干重复报账：按该记录电费金额估算。

如果缺少金额字段或金额不可解析，可追回金额记为 0，并在建议动作中提示补充金额口径。

### 压降机会金额

压降机会金额用于疑似优化空间：

- 高电价：当前电量乘以当前单价与参考单价差额。参考单价优先使用同地市、同区县、同供电方式中位数；不足样本时使用规则配置阈值。
- 转供电偏高：当前电量乘以转供电单价与参考单价差额。
- 异常增长：当前金额减去历史参考金额。历史参考金额优先使用上一账期金额；后续可扩展为近 3 期中位数。
- 疑似空载：按当前金额或连续空载周期金额合计估算。

压降机会金额只代表核查和优化空间，不直接等同于应追回金额。

## 技术设计

### 数据模型

第一版新增一张专题分析结果表，建议命名为 `analysis_opportunities`。字段包括：

- `id`
- `batch_id`
- `ledger_row_id`
- `domain`，第一版固定为 `electricity`
- `opportunity_code`
- `opportunity_type`
- `severity`
- `city`
- `district`
- `telecom_site_code`
- `telecom_site_name`
- `period`
- `meter_no`
- `current_amount`
- `reference_amount`
- `recoverable_amount`
- `saving_opportunity_amount`
- `confidence`
- `source_rule_ids_json`
- `message`
- `suggestion`
- `created_at`

该表作为专题分析输出，不替代 `issues`。同一条 `issues` 可以生成一条或多条机会记录；后续租费分析也写入同表，`domain` 区分专题。

### 分析服务

新增电费分析服务模块，建议命名为 `electricity_analysis.py`，职责如下：

- 从当前批次读取电费台账、稽核问题和原始字段。
- 复用现有电费规则命中结果生成确定性机会。
- 执行补充分析逻辑，例如高电价参考、环比增长、疑似空载。
- 计算可追回金额和压降机会金额。
- 写入或刷新 `analysis_opportunities`。
- 返回总览、分类和清单数据。

专题分析应设计为可重复运行。每次运行某批次电费分析前，先删除该批次 `domain = electricity` 的旧机会记录，再生成新结果。

### 接口

新增本地 API：

- `POST /api/batches/{batch_id}/electricity-analysis/run`：生成或刷新电费分析。
- `GET /api/batches/{batch_id}/electricity-analysis/summary`：读取总览。
- `GET /api/batches/{batch_id}/electricity-analysis/opportunities`：读取机会清单，支持地市、类型、风险、置信度筛选。
- `POST /api/batches/{batch_id}/electricity-analysis/export`：导出 Excel。

接口错误沿用现有服务风格，返回明确中文错误，例如批次不存在、批次未导入电费台账、尚未执行稽核、批次已归档等。

### 前端

新增“电费压降分析”导航入口。页面结构：

- 顶部操作区：当前批次、运行分析、导出 Excel。
- 总览卡片：电费总额、异常站址、可追回金额、压降机会金额。
- 分类统计：按异常类型展示数量、金额和高风险数量。
- 地市排行：按可追回金额和压降机会金额排序。
- 机会清单表格：支持筛选和搜索。

第一版不需要复杂图表，优先使用现有样式和表格组件，保持本地工作台的一致体验。

### 导出

新增电费压降机会清单 Excel，包含：

- 填写说明。
- 机会清单。
- 地市汇总。
- 异常分类汇总。

导出文件保存到现有 `exports/` 目录，文件名包含批次编号和“电费压降机会清单”。

## 错误处理

- 批次不存在：提示“批次不存在”。
- 缺少电费台账：提示“当前批次没有电费台账，无法生成电费压降分析”。
- 尚未稽核：提示“请先执行稽核，再生成电费压降分析”。
- 金额字段缺失：该条机会金额按 0 处理，保留机会记录并提示补充金额口径。
- 参考样本不足：不生成统计型高电价机会，或退回使用规则配置阈值，并降低置信度。
- 批次已归档：允许查看和导出历史分析结果，不允许刷新分析。

后端测试覆盖：

- 可追回金额计算。
- 压降机会金额计算。
- 重复运行时旧结果刷新。
- 批次不存在、未稽核、无电费台账等错误。
- 导出 Excel 包含必要 sheet 和字段。

前端测试或静态检查覆盖：

- 新导航入口加载。
- 无分析结果时的空状态。
- 运行分析后的总览和清单渲染。
- 筛选条件组合。
- 导出按钮错误提示。

## 分期

第一期：

- 新增专题分析结果表。
- 新增电费分析服务。
- 复用现有电费稽核问题生成机会清单。
- 实现总览、清单接口和 Excel 导出。
- 新增前端“电费压降分析”页面。

第二期：

- 补充环比、连续高电价、疑似空载和转供电偏高分析。
- 优化参考金额和置信度口径。
- 增加地市排行和分类统计细节。

第三期：

- 将机会清单接入整改回传或专题闭环。
- 抽象专题分析框架，复用到铁塔租费异常分析。
- 增加专题归档报表。
