# 下一批产品优化方案

## 背景

最近一轮已经完成电费压降、租费异常与通用问题状态之间的核查闭环。下一批不再优先增加新的分析类型，而是把现有能力从“功能可用”推进到“业务人员不容易走错、复核人员处理更快、状态口径更清楚”。

本方案基于 2026-07-17 的本地页面走查，覆盖：无批次工作台、新建批次后的流程引导、数据导入、稽核结果、电费压降分析和租费异常分析。由于走查环境没有代表性业务台账，本轮结论不包含大数据量下的列表性能、真实导入错误恢复和完整回传操作效率。

## 优化目标

- 降低首次使用者从新建批次到完成首次稽核的试错次数。
- 让 `returned` 与 `needs_review` 等不同业务状态在界面上可以被准确区分。
- 让复核人员进入专题页面后，首屏即可看到需要处理的线索，而不是先浏览大量零值指标。
- 阻止无数据、未生成分析或未完成预检时触发无效操作。
- 补齐筛选器、切换器和状态变化的可访问性语义。

## 优先级结论

### P0：状态口径与操作防错

#### 1. 拆分两个“待复核”状态

当前 `returned` 和 `needs_review` 在两个专题页面中都显示为“待复核”，筛选器中因此出现两个同名选项。用户无法判断一条记录是“刚回传、尚未判断”，还是“已填写核查信息、等待人工复核”。

统一为：

- `returned`：`已回传待确认`
- `needs_review`：`待人工复核`
- `still_invalid`：`仍需整改`
- `closed`：`已确认闭环`
- `not_required`：`无需整改`
- `resolved_by_reaudit`：`复审已解决`

状态标签、筛选器、工作台汇总、导出 Excel 和归档报表使用同一份前后端口径表。禁止在各页面分别维护显示文案。

验收标准：

- 任一筛选器中不存在同名但值不同的状态选项。
- 页面、Excel 与归档中的同一状态显示一致。
- 状态转换后的成功提示明确说明“从什么状态变成什么状态”。

#### 2. 增加前置条件与动作禁用原因

当前无台账批次仍可看到“执行稽核”“生成分析”“导出 Excel”等可点击主动作；数据导入页的“导入台账”也与“预检模板”同时处于可操作状态。下一批把所有关键动作改为显式状态门控：

- 未选择文件：禁用“预检模板”和“导入台账”。
- 未预检或预检失败：禁用“导入台账”，显示最近一次预检状态。
- 当前批次无对应台账：禁用“执行稽核”和“生成分析”，提供“去导入台账”入口。
- 尚未生成专题分析：禁用“导出 Excel”，提示先生成分析。
- 批次已归档：所有写操作保持禁用，并统一显示归档锁定说明。
- “覆盖当前批次”在提交前展示影响范围，包括将被替换的台账记录数和后续需重新执行的步骤。

验收标准：

- 用户不能仅依赖后端报错才知道动作不可执行。
- 每个禁用动作旁都有可见原因或下一步入口。
- 前端门控不替代后端校验；接口仍需拒绝越序操作。

### P1：专题复核效率

#### 3. 把专题页面首屏从“指标墙”改为“待办优先”

电费和租费页面当前分别在机会清单前展示 9 到 10 张指标卡。业务人员进入页面的主要任务是筛选和复核线索，首屏却看不到列表。

调整为：

- 首行仅保留 4 个主指标：机会/线索数、待人工复核、核实可追回、实际落实。
- 其余金额与状态指标收进“更多指标”折叠区。
- 主指标卡可点击并直接应用对应筛选。
- 首屏紧接“待处理清单”，默认按高风险、待人工复核、金额降序排列。
- 电费与租费页面复用同一布局与筛选组件，领域差异只体现在文案和字段。

验收标准：

- 在 1366×768 视口下无需滚动即可看到至少一条待处理记录或明确空状态。
- 点击指标卡后，筛选条件与结果数量同步变化。
- 两个专题页面的状态、筛选和核查操作位置一致。

#### 4. 增加复核队列与批量处理

在已有单条核查能力上增加“复核队列”：

- 支持按地市、异常类型、风险、状态和金额区间组合筛选。
- 支持保存当前筛选为本机常用视图。
- 支持勾选多条记录批量标记“仍需整改”或“无需整改”；确认闭环仍要求逐条核实金额与说明。
- 批量操作前展示影响条数，完成后提供成功、跳过和失败明细。

验收标准：

- 批量操作保持现有部分成功语义，失败项可定位和重试。
- 已归档批次不可进入批量编辑。
- 操作日志可追溯批量操作的来源、时间、目标状态和记录范围。

### P1：跨页面联动

#### 5. 在专项工作台增加专题待办

当前工作台能引导“导入—稽核—导出—回传—归档”，但专题分析与复核仍位于侧边导航的独立入口。新增“专题待办”区：

- 展示电费与租费的未生成、待处理、待人工复核、仍需整改数量。
- 每个数字可直接跳转到带对应筛选条件的专题页面。
- 下一步动作优先处理阻断归档的专题待办。
- 当通用问题与专题机会来自同一问题编号时，展示关联入口，避免两个页面重复判断。

验收标准：

- 工作台数字与专题页面相同筛选口径一致。
- 深链跳转后保留批次、领域和状态筛选。
- 没有专题台账或尚未生成分析时显示明确下一步，而不是一组零值指标。

### P2：易用性与可访问性

#### 6. 补齐控件名称和选择状态

页面结构走查显示，稽核结果页的多个下拉筛选缺少可访问名称；明细/聚合、状态快捷筛选和侧边导航的选中态主要依赖颜色表达。

改进项：

- 每个输入框与下拉框都使用显式 `label` 或等效可访问名称。
- 明细/聚合与快捷筛选使用 `aria-pressed` 表达选择状态。
- 当前导航使用 `aria-current="page"`。
- 关键异步结果通过可读状态区域播报，但避免整个主内容区频繁重复播报。
- 错误信息与对应字段建立关联，焦点移动到首个错误字段。
- 保持现有清晰的键盘焦点样式，并补充文件选择、原生下拉与专题核查表单的键盘走查。

验收标准：

- 仅使用键盘可以完成筛选、预检、查询与专题单条核查。
- 屏幕阅读器可以分辨每个筛选器的用途和当前选中的视图。
- 状态不只通过颜色区分。

## 建议实施顺序

### 第一阶段：口径与防错（2—3 个工作日）

1. 建立唯一状态展示映射并替换各页面重复实现。
2. 修复两个同名“待复核”筛选项。
3. 为导入、稽核、生成分析、导出和归档写清前置条件矩阵。
4. 同步完成前后端门控与回归测试。

### 第二阶段：专题页面重排（3—4 个工作日）

1. 抽取电费/租费共用的指标、筛选和待办布局。
2. 主指标精简为 4 个，其余收进折叠区。
3. 默认展示待处理队列并支持指标卡联动筛选。
4. 在工作台加入专题待办与深链。

### 第三阶段：批量复核与可访问性（3—5 个工作日）

1. 增加复核队列勾选、批量确认和结果反馈。
2. 完善控件名称、选择状态、错误焦点与键盘操作。
3. 使用代表性脱敏台账完成全流程验收与大列表性能检查。

## 暂不纳入本批

- 新增第三种专题分析。
- 用户账号、角色权限和多级审批。
- 云端同步或多人协作。
- 重做视觉品牌和大范围换肤。
- 修改现有机会测算算法；算法优化需要另行准备业务样本与准确率基线。

## 风险与验证要求

- 当前走查只有空状态和新建批次状态，专题列表的真实信息密度、长文本、金额分布和批量处理效率仍需脱敏样本验证。
- 状态文案调整会影响页面、Excel、归档和测试快照，应一次性统一，避免短期混用。
- 批量复核会放大误操作风险，必须保留影响预览、归档锁定、部分成功明细和完整操作日志。
- 页面截图无法证明完整的 WCAG 合规性；键盘、屏幕阅读器、缩放与高对比度模式需要单独验收。
