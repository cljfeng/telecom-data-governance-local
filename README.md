# Local Base Data Governance

本项目是省公司本地单机版基础数据治理工具，用于导入站址、铁塔租费、电费、发电费台账，执行稽核，导出地市整改包，并导入整改回传。

## Local Run

业务人员推荐使用独立可执行程序。开发人员或临时调试可以继续使用本地源码方式运行。

## 独立可执行程序打包

macOS/Linux 开发电脑上运行：

```bash
scripts/build_app.sh
```

生成文件：

```text
dist/zufeidianfei-governance
```

Windows 开发电脑上打开 PowerShell，进入项目目录后运行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\scripts\build_app.ps1
```

生成文件：

```text
dist\zufeidianfei-governance.exe
```

把 `dist/zufeidianfei-governance` 复制到业务电脑后，双击或在终端运行即可启动本地服务，并自动打开浏览器。默认数据仍保存在程序运行目录下的 `data/governance.sqlite3`，导出文件保存在 `exports/`。

macOS 下请运行 `dist/zufeidianfei-governance`。PyInstaller 构建过程中如出现 `build/` 目录或 `.pkg` 中间文件，它们不是交付给业务人员运行的程序。

如需指定工作目录：

```bash
dist/zufeidianfei-governance --workspace /Users/you/Documents/zufeidianfei
```

Windows 下对应命令：

```powershell
.\dist\zufeidianfei-governance.exe --workspace C:\zufeidianfei
```

## GitHub Actions 生成 Windows 程序

项目包含 `.github/workflows/build-windows-exe.yml`，可以在 GitHub 的 Windows runner 上自动生成 `.exe`。

手动生成步骤：

1. 打开 GitHub 仓库的 **Actions** 页面。
2. 选择 **Build Windows EXE**。
3. 点击 **Run workflow**。
4. 等待任务完成后，在本次 workflow run 的 **Artifacts** 下载 `zufeidianfei-governance-windows`。
5. 解压后得到 `zufeidianfei-governance.exe`，复制到 Windows 业务电脑运行。

推送到 `main` 分支且改动涉及 `src/`、打包脚本或 `pyproject.toml` 时，也会自动触发 Windows 打包。

## 源码本地运行

```bash
python -m venv .venv
. .venv/bin/activate
.venv/bin/python -m pip install -e ".[test]"
PYTHONPATH=src .venv/bin/python -m governance_app.server --workspace . --port 8765
```

也可以直接使用一键启动脚本：

```bash
scripts/start.sh
```

如需使用其他端口：

```bash
PORT=8877 scripts/start.sh
```

启动后在浏览器打开：

```text
http://127.0.0.1:8765
```

显式设置 `PYTHONPATH=src` 可以确保在尚未完成 editable install，或本地虚拟环境未加载 editable `.pth` 文件时仍能启动。

默认数据保存到 `data/governance.sqlite3`，导出文件保存到 `exports/`。不要把真实生产数据提交到 git。

## First Release Scope

- 支持四类台账模板导入：站址、铁塔租费、电费、发电费。
- 支持 SQLite 本地存储，保留导入批次和原始行。
- 支持基础稽核规则和稳定问题编号。
- 支持模板 sheet/字段别名兼容、行级预检错误提示和规则启停/阈值配置接口。
- 支持按地市导出整改问题包，也支持全省汇总单文件导出。
- 支持导入地市整改回传并更新问题状态。
- 支持专项工作台基础指标、问题分类统计、通报导出、本地数据库备份恢复和系统复位。

## Local Workbench

浏览器工作台已经按省公司本地专项治理流程组织：

- **专项工作台**：查看当前批次、流程步骤、下一步动作、关键指标和地市整改进度。
- **数据导入**：输入本机 Excel 模板路径，导入后自动生成并选中当前批次。
- **数据整理**：按台账类型、地市、站址编码查看导入后的原始台账数据，并按业务字段分组展开。
- **稽核结果**：执行当前批次稽核，按地市、台账、状态筛选问题，并支持人工关闭问题。
- **问题包导出**：可选择分地市单独文件，或全省汇总到一个 Excel 的单个 sheet，并将问题更新为待整改。
- **整改回传**：导入地市回传文件，展示匹配结果和地市整改进度。
- **分析报表**：展示问题统计和分类分析，支持一键导出稽核问题通报 Excel，并生成批次归档汇总 Excel。
- **归档前检查**：归档前提示批次状态、未闭环问题数量和阻断原因，避免误归档。
- **本地设置**：支持备份、恢复和系统复位。复位默认保留导出文件和备份文件，也可选择清理。

## 业务人员使用流程

1. 启动本地服务并打开 `http://127.0.0.1:8765`。
2. 进入“数据导入”，填写本机 Excel 台账文件完整路径，先点击“预检模板”。
3. 预检通过后点击“导入台账”，系统会自动生成并选中当前批次。
4. 进入“稽核结果”，点击“执行稽核”，查看问题清单和规则命中情况。
5. 进入“分析报表”，查看稽核问题统计、分类统计，必要时导出通报 Excel。
6. 进入“问题包导出”，选择分地市或全省汇总模式，导出的文件在 `exports/` 目录下。
7. 地市线下填写整改结果后，进入“整改回传”，填写回传文件完整路径并导入。
8. 回传后在“整改回传”或“专项工作台”查看地市整改进度。
9. 问题复核完成后进入“分析报表”，生成归档汇总；归档后的批次会被锁定，不再允许继续修改。

## 常见问题

- **文件路径怎么填？** 填写本机 Excel 文件的完整路径，例如 `/Users/you/Desktop/台账.xlsx`。最近预检或导入过的文件会出现在“最近文件”，点击“填入”可以自动回填。
- **导出文件在哪里？** 默认在工作区的 `exports/` 目录。地市整改包、导入错误明细和归档汇总都会写到该目录下。
- **为什么导出前提示需要先稽核？** 问题包来自稽核结果，必须先完成当前批次稽核才能导出。
- **模板表头有轻微差异怎么办？** 系统已经兼容常见 sheet 名和字段别名，例如“站址清单”“站址编码”“所属地市”等；预检会继续提示缺失字段、数字格式、日期格式、比例范围、枚举值和重复站址编码等问题。
- **为什么归档前提示批次未就绪？** 批次需要完成导出和回传流程后再归档；没有问题的批次执行导出后会直接进入可归档状态。
- **归档后为什么不能再改？** 归档代表该轮专项治理已形成正式留痕，系统会锁定批次，避免归档报表和后续修改不一致。
- **如何备份和恢复？** “分析报表”以外的备份恢复接口会使用本地 `backups/` 目录。备份文件只应保存在本机安全位置，不要提交到 git。
- **如何重新初始化？** 进入“本地设置”的“系统复位”，输入“复位”后执行。默认清除批次、台账、问题和回传记录，保留 `exports/` 和 `backups/`；如需彻底清理，可取消保留选项。

## 验收流程

1. 使用样例或脱敏台账执行一次“预检模板”，确认四类台账记录数能正常展示。
2. 正式导入同一台账，确认“专项工作台”显示当前批次和下一步动作。
3. 执行稽核，确认“稽核结果”能看到问题编号、地市、台账、规则名称、风险和状态。
4. 进入“数据整理”，按地市和站址编码查询四类台账，确认字段按业务分组展示。
5. 导出问题包，确认每个地市生成独立 Excel，且包含“规则编号”和“规则名称”。
6. 在整改包中填写“整改结果”“整改说明”“整改后值”，再导入回传。
7. 确认地市整改进度更新，问题进入待复核或已关闭等闭环状态。
8. 生成归档汇总，确认归档 Excel 包含归档总览、地市整改进度、问题清单和操作日志。
9. 重新尝试修改已归档批次的问题状态，确认系统拒绝修改。
